## 1. SERVICES D'UPSELL (avec commissions employés)

### Vue d'ensemble

Le système d'upsell transforme chaque employé en business developer. L'employé terrain identifie l'opportunité, la signale dans ServiceM8, et reçoit une commission automatique si le client accepte.

**Principe clé** : Les employés sont déjà sur place. Chaque job est une opportunité de vente additionnelle à coût marginal quasi-nul.

### 1.1 Fertilisation (Deep Root Feeding)

**Description**
Injection d'engrais liquide directement dans le système racinaire des cèdres pour améliorer la santé, la couleur et la densité du feuillage.

**Coût entreprise** : 15-25$ par haie (matériel + temps)
**Prix client** : 75-150$ par application (selon longueur de haie)
**Commission employé** : 15$ par upsell vendu
**Applications recommandées** : 3 par année (Avril, Mai, Juin)

**Saisonnalité** : Avril-Juin (croissance active)

**Pricing Formula**
```
0-15 pieds linéaires  : 75$
16-30 pieds           : 100$
31-50 pieds           : 125$
50+ pieds             : 150$
```

**Revenus projetés (scénario 2M$)**
- Base clients : 3,500 clients
- Taux d'adoption estimé : 15% = 525 clients
- Mix : 40% × 1 app, 40% × 2 apps, 20% × 3 apps
- Prix moyen par application : 110$
- Revenus : 525 × (0.4×1×110 + 0.4×2×110 + 0.2×3×110) = 525 × 176 = **92,400$/an**
- Coût : 525 × 1.6 apps × 20$ matériel = 16,800$
- Commissions : 525 × 1.6 apps × 15$ = 12,600$
- **Marge nette : 63,000$ (68%)**

### PROCESSUS TERRAIN — Comment l'employé upsell

**Étape 1 : Identification sur place**
L'employé effectue la taille régulière et identifie visuellement :
- Cèdres morts ou bruns → Remplacement
- Feuillage clairsemé/pâle → Fertilisation
- Signes de parasites (toiles, trous, brunissement) → Traitement
- Haie très négligée → Rabattage sévère
- Exposition au vent/route → Protection hivernale
- Base de haie dénudée → Paillis

**Étape 2 : Documentation dans ServiceM8**

```

**Étape 3 : Génération automatique de quote**
- Webhook ServiceM8 → API Haie Lite (`/api/upsell/flag`)
- Système calcule prix selon formule
- Quote auto-généré et envoyé par SMS dans les 2h

**Template SMS**
```
Bonjour [PRENOM],

[OPPORTUNITE]. Nous recommandons [SERVICE] pour [BENEFICE].

**Étape 4 : Conversion et commission**
- Client clique sur lien → Job créé dans ServiceM8
- Job complété → Système marque commission dans Supabase
- Paie suivante → Commission versée à l'employé

**Étape 5 : Tracking commission**
Dashboard employé dans l'app montre en temps réel :
- Upsells flaggés ce mois
- Upsells convertis
- Commissions accumulées (en attente de paie)

## 2. MODÈLE D'ABONNEMENT ANNUEL — "CLUB HAIE LITE"

### Philosophie

Le modèle d'abonnement transforme des transactions ponctuelles en revenus récurrents prévisibles. Il garantit :
1. **Cash flow pré-saison** (février-mars) avant même le début des opérations
2. **Taux de rétention 90%+** (vs 60-70% pour clients transactionnels)
3. **Planification opérationnelle** (nombre de crews requis connu à l'avance)
4. **Premium pricing** (clients paient plus annuellement mais perçoivent de la valeur)

### Structure en 3 tiers

```
┌─────────────────────────────────────────────────────────────────┐
│                     CLUB HAIE LITE                              │
│             Abonnement annuel avec privilèges               │
└─────────────────────────────────────────────────────────────────┘

┌──────────────┬──────────────┬──────────────┬──────────────────┐
│  ESSENTIEL   │   PREMIUM    │   PLATINE    │   COMPARAISON    │
├──────────────┼──────────────┼──────────────┼──────────────────┤
│   45$/mois   │   75$/mois   │  120$/mois   │  Pay-per-use:    │
│   540$/an    │   900$/an    │  1,440$/an   │  575$/taille     │
└──────────────┴──────────────┴──────────────┴──────────────────┘
```

### IMPLÉMENTATION TECHNIQUE

**Stack**
- **Stripe Billing** pour gestion abonnements et paiements récurrents
- **Supabase** pour tracking niveau membre, services utilisés, crédits
- **ServiceM8** pour tags clients et auto-scheduling
- **Twilio** pour rappels SMS pré-saison

**Workflow d'inscription**
```
Client visite haielite.com/club
    ↓
Sélectionne tier (Essentiel/Premium/Platine)
    ↓
Stripe Checkout (carte + autorisation récurrente)
    ↓
Webhook Stripe → API /api/subscriptions/create
    ↓
Supabase : Création enregistrement subscription
    ↓
ServiceM8 : Tag client "Club-Essentiel" (ou Premium/Platine)
    ↓
Email confirmation + bienvenue
    ↓
Février suivant : Cron job auto-schedule services inclus
```

**Gestion des services utilisés**
```sql
-- Tracking services inclus vs utilisés
SELECT
  client_uuid,
  plan,
  services_included,
  services_used,
  (services_included - services_used) AS services_remaining
FROM subscriptions
WHERE status = 'active';
```

**Auto-scheduling saisonnier (Février)**
Cron job tourne le 15 février :
```
1. Query tous les membres actifs
2. Pour chaque membre :
   - ESSENTIEL : Créer 1 job en avril (choix client printemps/automne)
   - PREMIUM : Créer 2 jobs (avril + septembre)
   - PLATINE : Créer 3 jobs (avril + juin + septembre)
   - + Jobs fertilisation/traitement selon tier
3. Jobs créés dans ServiceM8 avec priorité élevée
4. SMS envoyé : "Votre saison Haie Lite commence! Vos dates : ..."
```

**Gestion annulation/pause**
- Annulation avant renouvellement : remboursement prorata
- Pause hivernale : option de suspendre novembre-février (pas de facturation)
- Upgrade/downgrade : changement immédiat avec ajustement prorata

## 3. CONTRATS COMMERCIAUX (B2B)

### Opportunité

Le résidentiel est saisonnier et dispersé géographiquement. Le **commercial** offre :
- **Densité** : 1 site = revenus équivalents à 5-20 clients résidentiels
- **Récurrence** : Contrats multi-années (2-3 ans typique)
- **Prévisibilité** : Facturation programmée, pas de marketing requis
- **Marges** : Moins de déplacement = efficience accrue

---

### SEGMENTS B2B

#### 3.1 Condos et syndicats de copropriété

**Profil**
- Immeubles en copropriété avec haies de cèdres périmètre
- Gestionnaires immobiliers ou conseils d'administration
- Budget voté annuellement en assemblée générale
- Contrats généralement 2-3 ans avec renouvellement automatique

**Taille de contrat typique**
- Petit condo (12-24 unités) : 2,000-4,000$/an
- Moyen condo (25-60 unités) : 5,000-10,000$/an
- Grand complexe (60+ unités) : 12,000-25,000$/an

**Exemple — Condo Les Érables (48 unités, Brossard)**
```
Périmètre : 280 pieds linéaires de cèdres
Services :
- 2 tailles/an (mai + septembre)
- 1 fertilisation/an (mai)
- Remplacement cèdres morts (max 10/an)

Prix : 8,500$/an (contrat 3 ans)
Marge : 60% (peu de déplacement, jobs groupés)
```

**Prospection**
- **RGCQ** (Regroupement des gestionnaires et copropriétaires du Québec) — événements networking
- LinkedIn : Recherche "gestionnaire immobilier Rive-Sud"
- Référence clients résidentiels sur conseil d'administration
- Cold call : Liste syndicats via Registre foncier

**Documents requis**
- Preuve assurance responsabilité 2M$
- WCB (CNESST)
- Soumission détaillée avec photos/plan du site
- Références clients commerciaux

---

#### 3.2 Municipalités

**Profil**
- Parcs municipaux, terre-pleins, entrées de ville
- Processus d'appel d'offres formel (SEAO pour >25k$)
- Contrats 1-3 ans, parfois avec options de renouvellement
- Paiement lent (60-90 jours) mais garanti

**Taille de contrat typique**
- Petite municipalité (<10k habitants) : 5,000-15,000$/an
- Municipalité moyenne : 20,000-50,000$/an

**Exemple — Ville de Saint-Bruno-de-Montarville**
```
Sites :
- Parc des Grands-Jardins (120 pieds)
- Entrée autoroute 30 (200 pieds)
- Hôtel de ville (80 pieds)

Services :
- 2 tailles/an (juin + septembre)
- Nettoyage résidus inclus

Prix : 12,000$/an (contrat 2 ans + option 1 an)
Marge : 55% (sites accessibles, haute visibilité = marketing gratuit)
```

**Prospection**
- **COMAQ** (Corporation des officiers municipaux en aménagement) — congrès annuel
- **SEAO** (Système électronique d'appel d'offres) — surveillance catégorie "espaces verts"
- Appel direct aux directeurs travaux publics
- Partenariat avec paysagistes municipaux (sous-traitance)

**Pièges à éviter**
- Garanties de performance exigeantes (pénalités retard)
- Assurances spécifiques requises
- Prix ultra-compétitifs (course au moins-disant)

#### 3.3 Entreprises d'aménagement paysager (sous-traitance)

**Profil**
- Paysagistes généralistes qui sous-traitent la taille de haies (spécialité)
- Avantage : Flux constant de jobs, facturation B2B rapide
- Volume élevé mais marges réduites (partage avec donneur d'ordre)

**Taille de contrat typique**
- Petit paysagiste : 15,000-30,000$/an
- Gros paysagiste : 50,000-100,000$/an

**Exemple — Partenariat avec Paysages Nature Plus**
```
Deal : Haie Lite devient fournisseur exclusif taille de cèdres
Volume : ~150 jobs résidentiels/an passés par Nature Plus
Prix : 70% du prix client final (Nature Plus garde 30%)

Revenus : 150 × 575$ × 70% = 60,375$/an
Marge : 40% (réduite vs direct mais zéro marketing requis)
```

**Prospection**
- AAPQ (Association des architectes paysagistes du Québec)
- LinkedIn : Messagerie paysagistes Rive-Sud/Montérégie
- Offre "White label" : Haie Lite fait le job, paysagiste prend le crédit

#### 3.4 Centres commerciaux et bureaux

**Profil**

-- ============================================================
-- REVENUE ANALYTICS (Materialized View)
-- Vue agrégée pour reporting revenus par source
-- ============================================================

CREATE MATERIALIZED VIEW revenue_by_source AS
SELECT
  DATE_TRUNC('month', created_at) AS month,

  -- Revenue sources
  SUM(CASE WHEN source = 'base_jobs' THEN amount ELSE 0 END) AS base_jobs_revenue,
  SUM(CASE WHEN source = 'upsells' THEN amount ELSE 0 END) AS upsells_revenue,
  SUM(CASE WHEN source = 'subscriptions' THEN amount ELSE 0 END) AS subscriptions_revenue,
  SUM(CASE WHEN source = 'commercial' THEN amount ELSE 0 END) AS commercial_revenue,
  SUM(CASE WHEN source = 'referrals' THEN amount ELSE 0 END) AS referrals_revenue,
  SUM(CASE WHEN source = 'reactivation' THEN amount ELSE 0 END) AS reactivation_revenue,

  -- Totals
  SUM(amount) AS total_revenue,
  COUNT(*) AS transaction_count

FROM (
  -- Base jobs (ServiceM8 sync)
  SELECT
    created_at,
    'base_jobs' AS source,
    total_amount AS amount
  FROM jobs
  WHERE status = 'completed'

  UNION ALL

  -- Upsells convertis
  SELECT
    completed_at AS created_at,
    'upsells' AS source,
    estimated_value AS amount
  FROM upsell_opportunities
  WHERE status = 'completed'

  UNION ALL

  -- Subscriptions (revenus mensuels)
  SELECT
    current_period_start AS created_at,
    'subscriptions' AS source,
    monthly_amount AS amount
  FROM subscriptions
  WHERE status = 'active'

  UNION ALL

  -- Commercial contracts (pro-rata mensuel)
  SELECT
    DATE_TRUNC('month', CURRENT_DATE) AS created_at,
    'commercial' AS source,
    annual_value / 12 AS amount
  FROM commercial_contracts
  WHERE status = 'active'

  UNION ALL

  -- Referrals (revenus jobs référés)
  SELECT
    converted_at AS created_at,
    'referrals' AS source,
    conversion_value AS amount
  FROM referrals
  WHERE status = 'converted'

  UNION ALL

  -- Reactivations
  SELECT
    converted_at AS created_at,
    'reactivation' AS source,
    conversion_value AS amount
  FROM reactivation_campaigns
  WHERE status = 'converted'

) AS all_revenue

GROUP BY DATE_TRUNC('month', created_at)
ORDER BY month DESC;

-- Refresh daily via cron
CREATE INDEX idx_revenue_by_source_month ON revenue_by_source(month DESC);

// Flag upsell opportunity from field

import { NextRequest, NextResponse } from 'next/server';
import { createClient } from '@/lib/supabase/server';
import { uploadPhoto } from '@/lib/storage';
import { calculateUpsellPrice } from '@/lib/pricing';

export async function POST(req: NextRequest) {
  const supabase = createClient();

  const {
    job_uuid,
    employee_id,
    client_uuid,
    service_type,
    description,
    photo_base64,
    quantity,
    urgency
  } = await req.json();

  // Upload photo to Supabase Storage
  const photo_url = await uploadPhoto(photo_base64, `upsells/${job_uuid}`);

  // Calculate estimated price based on service type
  const estimated_value = calculateUpsellPrice(service_type, quantity);

  // Get commission amount from lookup table
  const commission_amount = COMMISSION_TABLE[service_type];

  // Insert opportunity
  const { data, error } = await supabase
    .from('upsell_opportunities')
    .insert({
      job_uuid,
      employee_id,
      client_uuid,
      service_type,
      description,
      photo_url,
      quantity,
      estimated_value,
      commission_amount,
      urgency,
      status: 'flagged'
    })
    .select()
    .single();

  if (error) {
    return NextResponse.json({ error: error.message }, { status: 500 });
  }

  // Trigger quote generation (async job queue)
  await fetch('/api/upsell/quote', {
    method: 'POST',
    body: JSON.stringify({ opportunity_id: data.id })
  });

  return NextResponse.json({
    success: true,
    opportunity: data,
    message: 'Opportunité enregistrée. Quote sera envoyé au client dans 2h.'
  });
}

const COMMISSION_TABLE = {
  fertilisation: 15,
  pest_treatment: 20,
  winter_protection: 25,
  rejuvenation: 50,
  cedar_replacement: 10,
  mulching: 15
};

// Stripe webhook handler for subscription events

import { NextRequest, NextResponse } from 'next/server';
import { headers } from 'next/headers';
import Stripe from 'stripe';
import { createClient } from '@/lib/supabase/server';

const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!);
const webhookSecret = process.env.STRIPE_WEBHOOK_SECRET!;

export async function POST(req: NextRequest) {
  const body = await req.text();
  const signature = headers().get('stripe-signature')!;

  let event: Stripe.Event;

  try {
    event = stripe.webhooks.constructEvent(body, signature, webhookSecret);
  } catch (err: any) {
    console.error('Webhook signature verification failed:', err.message);
    return NextResponse.json({ error: 'Invalid signature' }, { status: 400 });
  }

  const supabase = createClient();

  switch (event.type) {
    case 'customer.subscription.created':
      const created = event.data.object as Stripe.Subscription;
      await handleSubscriptionCreated(supabase, created);
      break;

    case 'customer.subscription.updated':
      const updated = event.data.object as Stripe.Subscription;
      await handleSubscriptionUpdated(supabase, updated);
      break;

    case 'customer.subscription.deleted':
      const deleted = event.data.object as Stripe.Subscription;
      await handleSubscriptionDeleted(supabase, deleted);
      break;

    case 'invoice.payment_succeeded':
      const invoice = event.data.object as Stripe.Invoice;
      await handlePaymentSucceeded(supabase, invoice);
      break;

    case 'invoice.payment_failed':
      const failedInvoice = event.data.object as Stripe.Invoice;
      await handlePaymentFailed(supabase, failedInvoice);
      break;
  }

  return NextResponse.json({ received: true });
}

async function handleSubscriptionCreated(supabase: any, subscription: Stripe.Subscription) {
  const plan = subscription.metadata.plan; // 'essentiel' | 'premium' | 'platine'
  const client_uuid = subscription.metadata.client_uuid;

  const planConfig = {
    essentiel: { services: 1, fertilisations: 0, monthly: 45, annual: 540 },
    premium: { services: 2, fertilisations: 1, monthly: 75, annual: 900 },
    platine: { services: 3, fertilisations: 3, monthly: 120, annual: 1440 }
  };

  const config = planConfig[plan as keyof typeof planConfig];

  await supabase.from('subscriptions').insert({
    client_uuid,
    stripe_subscription_id: subscription.id,
    stripe_customer_id: subscription.customer,
    plan,
    monthly_amount: config.monthly,
    annual_amount: config.annual,
    services_included: config.services,
    fertilisations_included: config.fertilisations,
    status: 'active',
    current_period_start: new Date(subscription.current_period_start * 1000),
    current_period_end: new Date(subscription.current_period_end * 1000)
  });

  // Tag client in ServiceM8
  await tagServiceM8Contact(client_uuid, `Club-${plan}`);

  // Send welcome email
  await sendWelcomeEmail(client_uuid, plan);
}

import { NextRequest, NextResponse } from 'next/server';
