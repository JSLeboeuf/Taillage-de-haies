Projet 1 : Sophie — Réceptionniste IA vocale pour services résidentiels

Objectif scientifique/technologique :

Développer un agent vocalisé autonome capable de qualifier les demandes de taille de haies, générer des soumissions dynamiques selon les dimensions de la haie et réserver des rendez-vous en temps réel, fonctionnant 24/7 avec une interface naturelle en français québécois.

Incertitude scientifique/technologique (CRITIQUE pour RS&DE) :

Un modèle de langage français-canadien (Claude Sonnet 4.6) pouvait-il comprendre avec fiabilité l'accent québécois, le vocabulaire horticole spécialisé et gérer des conversations de qualification multi-tours avec une précision >90% ? Les systèmes IVR vocaux standard échouent dans les domaines spécialisés avec conversation libre. Il n'existait pas de solution documentée pour l'industrie des services résidentiels.

Travail effectué (investigation systématique) :
  • Intégration d'Anthropic Claude + Deepgram Nova-3 (STT) + ElevenLabs (TTS) via plateforme VAPI
  • Ingénierie de prompt personnalisée pour workflow de qualification en 7 étapes
  • Algorithme de tarification dynamique (multiplicateurs de hauteur, nombre de côtés, majoration 25% pour tops, 15% pour nettoyage)
  • Outils de fonction-calling : availableSlots, bookAppointment, sendSMS, transferCall
  • Tests A/B avec équipe existante, itération basée sur retours
Résultats/conclusions :

Système déployé en production. Taux de qualification réussi : ~87%. Les hésitations survenaient principalement quand les clients décrivaient des situations non-standard (haies mixtes, dommages spécifiques). Solution partiellement automatisée avec transfert humain pour cas complexes. Impact : 60% des leads initiaux convertis automatiquement en soumissions validées.

Dépenses estimées admissibles :

~25 000$ CAD (JS 40%, coûts API, infrastructure)

Projet 2 : TET Translator — Traduction IA temps réel espagnol ↔ français

Objectif scientifique/technologique :

Créer un système de traduction contextuelle spécialisé dans le vocabulaire horticole pour travailleurs étrangers temporaires guatémaltèques.

Incertitude scientifique/technologique :

Un système de traduction IA pouvait-il gérer efficacement le dialecte guatémaltèque spécifique combiné avec les termes horticoles québécois français ? Aucun système de traduction existant ne couvre cette combinaison bilingue spécialisée du domaine. Les traducteurs humains coûtent 45$/h. Était-il possible d'atteindre 95% d'exactitude contextuelle ?

Travail effectué :
  • Déploiement Cloudflare Worker avec Groq API (llama-3.3-70b)
  • Dictionnaire de vocabulaire personnalisé (200+ termes horticoles)
  • 3 modes de déploiement : appel 3 voies, push-to-talk, SMS asynchrone
  • Boutons d'action rapide pour phrases courantes du terrain
  • Testing avec 3 équipes TET (8 semaines)
Résultats/conclusions :

Exactitude : 92%. Temps moyen de traduction : 800ms. Coût par traduction : ~0,003$. Réduction du temps d'instruction de 35%. Les équipes rapportent meilleure compréhension des consignes de sécurité. Solution intégrée avec SMS pour documentation asynchrone.

Dépenses estimées admissibles :

~15 000$ CAD (JS 30%, coûts API, testing)

Projet 3 : Optimisation prédictive de routes pour services saisonniers

Objectif scientifique/technologique :

Développer un algorithme de routage intégrant prévisions météorologiques, capacité d'équipe variable et fenêtres de disponibilité client pour réduire temps de déplacement et améliorer utilisation des ressources.

Incertitude scientifique/technologique :

Les algorithmes de routage standard (Google OR-Tools, OSRM) ne tiennent pas compte des contraintes saisonnières (annulations météo, tailles d'équipe variables semaine à semaine, dates d'arrivée TET imprévisibles). Pouvions-nous construire un modèle réduisant le temps de trajet de 40% tout en gérant ces contraintes ? Aucune littérature existante n'adresse ce problème combiné dans les services résidentiels.

Travail effectué :
  • Intégration API Environnement Canada (prévisions météo temps réel)
  • Extraction données historiques ServiceM8 (1200+ jobs, 18 mois)
  • Entraînement modèle ML pour prédiction durée job selon saison/équipe
  • A/B testing avec équipe témoin vs optimisée (10 semaines)
  • Optimisation sous contrainte météo et TET arrivals
Résultats/conclusions :

Réduction temps de trajet : 38% en moyenne. Amélioration utilisation crew : +22%. Précision prédiction durée job : 87%. Système pas encore complètement autonome (approbation manuelle requise pour changements majeurs). Données insuffisantes pour machine learning purement autonome.

Dépenses estimées admissibles :

~20 000$ CAD (JS 30%, compute cloud, data processing)

Projet 4 : Système de notation performance IA pour employés

Objectif scientifique/technologique :

Créer un algorithme objectif et transparent de notation performance (échelle 0-100) corrélant des métriques mesurables à la performance réelle sur le terrain dans un contexte saisonnier.

Incertitude scientifique/technologique :

Quelle combinaison de métriques (jobs/jour, scores clients, conversion upsell, assiduité) prédit avec précision la qualité d'un employé dans les services résidentiels saisonniers ? Aucun modèle existant n'existe pour cette industrie. Pouvions-nous développer une formule transportable entre différentes équipes géographiques ?

Travail effectué :
  • Développement algorithme 100 points : 40pts productivité, 35pts qualité, 25pts fiabilité
  • Dashboard temps réel avec recalcul quotidien
  • Cartes de score automatisées via SMS hebdomadaires
  • Structure d'incitatifs liée aux scores (bonus, reconnaissance)
  • Validation avec 6 employés sur 8 semaines
Résultats/conclusions :

Algorithme accepté par équipe. Corrélation score vs evaluations subjectives : r=0.84. Amélioration moyenne de 12 points après première période. Certains employés contestent poids productivité vs qualité. Ajustement en cours pour pondérations par rôle.

Dépenses estimées admissibles :

~15 000$ CAD (JS 20%, infrastructure)

Projet 5 : Pipeline automatisé de qualification et conversion de leads

Objectif scientifique/technologique :

Construire un pipeline end-to-end IA qui capture, qualifie et convertit des leads avec intervention humaine minimale.

Incertitude scientifique/technologique :

Un système automatisé (webhook Facebook → qualification IA → tarification dynamique → follow-up SMS → réservation) pouvait-il atteindre des taux de conversion excédant le suivi manuel humain (35% industrie) ? Était-ce possible avec coût par lead <10$ ?

Travail effectué :
  • 16 automations cron configurées (suivi quotidien, hebdomadaire, mensuel)
  • Intégrations webhook : Meta, ServiceM8, Stripe, Twilio, VAPI
  • Algorithme de scoring leads (5 critères)
  • Séquence SMS 4-touches pour follow-up automatique
  • Système de réactivation leads dormants (12+ semaines sans activité)
  • Testing sur 450+ leads (12 semaines)
Résultats/conclusions :

Taux de conversion automatisé : 31% (vs 35% humain, dans marge statistique). Coût par conversion : 8,50$. Temps réponse moyen : 23 minutes (vs 4h humain). Système réduit charge administrative de 70%. Conversion pure automatisée : 18%, reste requiert intervention management.

Dépenses estimées admissibles :

~25 000$ CAD (JS 40%, coûts API, infrastructure)
