# Add-on Structure Documentation

## Overview

ServiceM8 add-ons consist of five key components working together to extend platform functionality.

## Core Components

Add-ons are composed of the following parts:

1. **Add-on Manifest** - Configuration file defining add-on properties and settings
2. **Add-on Assets** - Images, JavaScript files, and other static resources
3. **Client Library** - JavaScript SDK for client-side interactions
4. **Serverless Functions or Web Service** - Backend logic (Lambda Function or Web Service hosting)
5. **API Connectivity** - Integration with ServiceM8 REST APIs

## Architecture

The documentation includes a visual diagram showing how these components interconnect within the add-on ecosystem. Each element plays a specific role in enabling add-ons to integrate with and extend ServiceM8's core functionality.
