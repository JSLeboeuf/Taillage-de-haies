# ServiceM8 Add-on SDK Examples

## Overview

The ServiceM8 developer documentation provides guidance on extending the platform through Add-on SDK examples. These samples demonstrate core SDK capabilities and serve as starting points for custom development.

## Available Examples

### Hello World Add-on

The foundational example demonstrates minimum requirements for SDK integration. It introduces:
- Button integration on Job Card (online interface)
- HTML rendering for ServiceM8 UI
- Standard CSS includes
- Client-side JavaScript SDK usage

### Hello World Add-on (Python Version)

"This example is identical to the standard (NodeJS) Hello World Add-on, but it's written in Python instead."

### App Actions Add-on

Mobile-focused implementation showing:
- Job Card button placement on mobile app
- UI rendering with ServiceM8 styling
- CSS integration approaches

### Weather Add-on

Demonstrates external API integration with OpenWeatherMap data:
- OAuth token authentication
- REST API record reading
- Filter parameter implementation
- Third-party API consumption
- Dynamic HTML/CSS rendering

### Pool Calculator Add-on

Practical tool showcasing form-based workflows:
- User input collection interface
- Calculation logic integration
- Job Diary note posting

*Note: Example calculations use placeholder values for demonstration purposes.*

### Showcase Add-on

Comprehensive feature demonstration combining:
- Online and mobile button placement
- UI rendering patterns
- JavaScript SDK implementation
- OAuth authentication
- REST API integration

### Webhook Add-on

Event-driven pattern example covering:
- Webhook manifest registration
- Status change notifications
- REST API record queries
- File attachment uploads
