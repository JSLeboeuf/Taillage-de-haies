# Feed API Overview

## Purpose

The activity feed serves as a centralized communication hub within ServiceM8 accounts. It enables organizations to share important updates while being accessible to all staff members within an account.

**Key Use Cases:**
- Reporting functionality
- Customer feedback and reviews
- Booking notices

## Important Considerations

The documentation emphasizes that "messages are targeted (and not sensitive) to the entire organisation," indicating this is a shared channel meant for organization-wide communication rather than private messaging.

## API Implementation

The Feed API provides endpoints for posting updates to the activity feed. Developers integrating with ServiceM8 can leverage this API to automatically publish relevant information into the shared communication channel.
