   - Configs:

#### B. CORE WEB VITALS SPECIFIQUES

### Configurations Elementor Essentielles

**Dashboard Elementor > Settings > Optimisations:**

1. **Optimized DOM Output** ✓ ON
   - Réduit HTML générée par Elementor (~30% reduction)
   - Default: OFF → Changer à ON

2. **Asset Loading** ✓ "Async" or "Defer"
   - Default: Async
   - Keep: Async (charge JS en parallèle, bloque pas render)

3. **Improved Asset Loading** ✓ ON
   - Charge uniquement CSS/JS nécessaires par page
   - Default: OFF → Changer à ON

4. **CSS Print Media** ✓ OFF
   - Évite media-print CSS loading
   - Default: depends → Vérifier OFF

5. **Font Awesome Inline** ✓ ON
   - Intègre FA icons directement (évite requête externe)

6. **Google Fonts Inline** ✓ ON
   - Self-host Google Fonts (au lieu de requête Google CDN)
   - Important pour diminuer latency

7. **Disable Elementor Google Fonts** ✓ ON (si Self-Hosted via plugin)
   - After installing "Embed Google Web Fonts"
   - Utilise versions self-hosted (+ rapide)

8. **Lightbox** ✓ "Disable" if not used
   - Si galerie (avant/après) utilise native lightbox, laisser
   - Sinon: Disable pour réduire JS

### Actions Correctives Courantes

**Erreur: "Discovered - currently not indexed"**
- Cause: Page trouvée mais pas encore crawlée/indexée
- Fix: Cliquer "Request Indexation" (GSC > URL Inspection)
- Timeline: 24-72h

**Erreur: "Blocked by robots.txt"**
- Cause: WordPress robots.txt refuse crawler
- Fix: GSC > Settings > Check robots.txt, remove blocks for important pages

**Erreur: "Noindex tag"**
- Cause: Page a tag `<meta name="robots" content="noindex">`
- Fix: Elementor > Page Settings > Visibility → Remove noindex

**Erreur: "404 Not Found"**
- Cause: Lien cassé = 404 error
- Fix: Réparer URL ou redirect 301 vers page correcte
